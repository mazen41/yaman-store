# 🎁 Loyalty Cards System - نظام بطاقات الهدية والولاء

## 📋 Overview - نظرة عامة

نظام متكامل لإدارة بطاقات الهدية وبرامج الولاء يتيح للعملاء شراء بطاقات بأرصدة محددة واستخدامها في المشتريات مع إمكانية الحصول على مكافآت وعروض ترويجية.

---

## 🗄️ Database Structure - هيكل قاعدة البيانات

### **1. loyalty_cards - جدول البطاقات**

```sql
- id (Primary Key)
- card_number (رقم البطاقة - فريد)
- card_password (كلمة المرور - مشفرة)
- card_type (نوع البطاقة: gift/loyalty/promotional)
- initial_balance (الرصيد الأولي)
- current_balance (الرصيد الحالي)
- bonus_balance (رصيد المكافأة)
- total_spent (إجمالي الإنفاق)
- customer_id (ربط بالعميل - اختياري)
- customer_name (اسم العميل)
- customer_phone (رقم الهاتف)
- customer_email (البريد الإلكتروني)
- status (الحالة: active/inactive/expired/blocked)
- activation_date (تاريخ التفعيل)
- expiry_date (تاريخ الانتهاء)
- last_used_date (آخر استخدام)
- notes (ملاحظات)
- created_by (المستخدم المنشئ)
- is_active (نشطة/محذوفة)
- created_at, updated_at
```

### **2. loyalty_card_transactions - جدول المعاملات**

```sql
- id (Primary Key)
- card_id (Foreign Key -> loyalty_cards)
- transaction_type (نوع المعاملة):
  * purchase (شراء/إضافة رصيد)
  * refund (استرجاع)
  * transfer_in (تحويل وارد)
  * transfer_out (تحويل صادر)
  * bonus (مكافأة)
  * adjustment (تعديل)
- amount (المبلغ)
- balance_before (الرصيد قبل)
- balance_after (الرصيد بعد)
- order_id (ربط بطلب الشراء - اختياري)
- reference_number (رقم مرجعي)
- description (الوصف)
- transfer_to_card_id (البطاقة المحول إليها)
- created_by (المستخدم)
- transaction_date (تاريخ المعاملة)
```

### **3. loyalty_card_goals - جدول الأهداف**

```sql
- id (Primary Key)
- card_id (Foreign Key -> loyalty_cards)
- goal_name (اسم الهدف)
- goal_description (وصف الهدف)
- target_amount (المبلغ المستهدف)
- current_amount (المبلغ الحالي)
- goal_status (حالة الهدف: active/completed/cancelled)
- start_date (تاريخ البداية)
- target_date (تاريخ الهدف)
- completed_date (تاريخ الإنجاز)
- notes (ملاحظات)
- created_at, updated_at
```

### **4. loyalty_card_promotions - جدول العروض الترويجية**

```sql
- id (Primary Key)
- promo_code (كود العرض - فريد)
- promo_name (اسم العرض)
- promo_description (وصف العرض)
- promo_type (نوع العرض):
  * percentage (نسبة مئوية)
  * fixed_amount (مبلغ ثابت)
  * bonus_balance (رصيد إضافي)
- promo_value (قيمة العرض)
- min_purchase_amount (الحد الأدنى للشراء)
- max_discount_amount (الحد الأقصى للخصم)
- usage_limit (عدد مرات الاستخدام)
- used_count (عدد مرات الاستخدام الفعلي)
- start_date (تاريخ البداية)
- end_date (تاريخ النهاية)
- status (الحالة: active/inactive/expired)
- applicable_to (ينطبق على: all/specific_cards/new_cards)
- created_by (المستخدم المنشئ)
- created_at, updated_at
```

### **5. loyalty_card_promo_usage - جدول استخدام العروض**

```sql
- id (Primary Key)
- card_id (Foreign Key -> loyalty_cards)
- promo_id (Foreign Key -> loyalty_card_promotions)
- order_id (ربط بطلب الشراء)
- discount_amount (مبلغ الخصم)
- used_at (تاريخ الاستخدام)
```

## 🚀 How It Works - كيف يعمل النظام

### **السيناريو 1: شراء بطاقة هدية عادية**

```
1. العميل يشتري بطاقة بـ 500 ريال
2. النظام ينشئ بطاقة برقم فريد (GIFT-2025-001)
3. يتم تعيين كلمة مرور للبطاقة
4. الرصيد الأولي: 500 ريال
5. الرصيد الحالي: 500 ريال
6. يمكن للعميل استخدام البطاقة في المشتريات
```

### **السيناريو 2: شراء بطاقة مع عرض ترويجي**

```
1. يوجد عرض: "اشتري بـ 900 ريال واحصل على 100 ريال إضافية"
2. العميل يشتري بطاقة بـ 900 ريال
3. النظام يطبق العرض تلقائياً
4. الرصيد الأولي: 900 ريال
5. رصيد المكافأة: 100 ريال
6. الرصيد الحالي: 1000 ريال
7. يتم تسجيل استخدام العرض
```

### **السيناريو 3: تحويل رصيد بين بطاقتين**

```
1. البطاقة A لديها رصيد 500 ريال
2. البطاقة B لديها رصيد 200 ريال
3. تحويل 100 ريال من A إلى B
4. النظام يسجل معاملتين:
   - transfer_out من البطاقة A (-100)
   - transfer_in للبطاقة B (+100)
5. الرصيد النهائي:
   - البطاقة A: 400 ريال
   - البطاقة B: 300 ريال
```

### **السيناريو 4: استخدام البطاقة في طلب شراء**

```
1. العميل يقدم رقم البطاقة وكلمة المرور
2. النظام يتحقق من صحة البيانات
3. يتحقق من الرصيد المتاح
4. يتم خصم مبلغ الطلب من الرصيد
5. يتم تسجيل المعاملة مع ربطها بالطلب
6. يتم تحديث total_spent
7. يتم تحديث last_used_date
```

### **السيناريو 5: تحديد هدف ادخاري**

```
1. العميل يحدد هدف: "شراء جهاز بـ 2000 ريال"
2. يضيف رصيد شهرياً للبطاقة
3. النظام يتتبع التقدم نحو الهدف
4. عند الوصول للمبلغ المستهدف:
   - يتم تحديث goal_status إلى "completed"
   - يتم تسجيل completed_date
   - يمكن إرسال إشعار للعميل
```

## 📊 Use Cases - حالات الاستخدام

### **1. بطاقات الهدايا للمناسبات**
```
- شركة تبيع بطاقات هدايا بقيم مختلفة (100، 500، 1000 ريال)
- العملاء يشترون البطاقات كهدايا
- المستلم يستخدم البطاقة في المشتريات
- يمكن تتبع استخدام كل بطاقة
```

### **2. برنامج ولاء للعملاء**
```
- العميل يحصل على بطاقة ولاء
- مع كل عملية شراء يحصل على نقاط (رصيد)
- يمكن استخدام النقاط في مشتريات مستقبلية
- عروض خاصة لحاملي بطاقات الولاء
```

### **3. حملات ترويجية موسمية**
```
- عرض الصيف: اشتري بـ 900 واحصل على 1000
- بطاقات ترويجية لفترة محدودة
- تتبع نجاح الحملة من خلال الإحصائيات
- تحديد ROI للحملة
```

### **4. بطاقات الشركات**
```
- شركة تشتري بطاقات لموظفيها
- كل موظف يحصل على بطاقة برصيد محدد
- يمكن للشركة تتبع إنفاق كل موظف
- إعادة شحن البطاقات شهرياً
```

### **5. برنامج الادخار**
```
- العميل يحدد هدف ادخاري
- يضيف مبالغ دورية للبطاقة
- النظام يتتبع التقدم
- عند الوصول للهدف يمكن استخدام الرصيد
```

## 🔐 Security Features - الأمان

### **1. حماية البطاقات**
```php
- كلمات المرور مشفرة باستخدام password_hash()
- التحقق من كلمة المرور عند كل استخدام
- إمكانية حظر البطاقات المشبوهة
- تتبع جميع المعاملات مع تسجيل المستخدم
```

### **2. منع الاحتيال**
```php
- التحقق من الرصيد قبل كل معاملة
- تسجيل IP address لكل معاملة (يمكن إضافته)
- حد أقصى لعدد محاولات إدخال كلمة المرور الخاطئة
- إشعارات عند معاملات كبيرة
```

### **3. Audit Trail**
```php
- سجل كامل لجميع المعاملات
- تسجيل المستخدم الذي أجرى كل عملية
- تسجيل التاريخ والوقت لكل معاملة
- الرصيد قبل وبعد كل معاملة
```
